# S0 故事引擎 · Production Prompt v1

输出 `S0_story_engine.md`。内容必须来自用户提供的**素材卡**、大纲与小说摘录，**严禁**编造无关故事、角色或世界观。

**禁止**独立枚举名场面清单——名场面只存在于 P1；本阶段用 `card_id` 引用并裁决。

## 必须遵守的 Markdown 骨架（标题字面量不可改）

```markdown
# 故事引擎 · {作品名}

## 一句话引擎

> **{一句话，可裁决取舍}**

## 四台核心发动机（核心爽点）

### 引擎 1：{名称}
- **爽点机制**：…
- **代表节点**：evt_001 / mk_002（引用素材卡 id）
- **为什么不能压缩**：…

### 引擎 2：…
### 引擎 3：…
### 引擎 4：…

## 素材裁决表

| card_id | 裁决 | 理由 | 服务引擎 |
|---|---|---|---|
| mk_001 | 保留 | … | 引擎1 |
| red_003 | 删除 | … | — |
| evt_005 | 合并→mk_002 | … | 引擎1 |

（覆盖 P1 全部 mk_* 卡，每条有且仅有一条记录）

## 可删 / 可合并支线（做减法的对象）

- red_* 卡及裁决为删除/合并的条目（不自由发挥新名场面）
```

## 内容要求

- 四台发动机各 ≥3 代表节点，写清供能机制（不是剧情摘要）
- 编号必须用 `引擎 1`…`引擎 4`
- 素材裁决表必须覆盖所有 `mk_*` 卡；裁决为「保留」或「合并」的 mk 进入下游必保索引
- 一句话引擎锋利、可裁决取舍

只输出 Markdown，不要输出解释性前言。
