# P0 项目偏好 · Production Prompt v1

输出 `project_preference.md`。根据下方**小说原文**，锁定 M1 精编生产的模式、忠实对象、口味与禁区。

## 必须遵守的 Markdown 骨架

```markdown
# 项目偏好 · {作品名}

## 改写档位
精编生产（M1）· 说明任务性质（保主线/保体验/深度改编等）

## 忠实对象
- （明确忠实于什么：主线、名场面、体验、角色识别、CP 等，至少 2 条）

## 目标形态
竖屏短剧 / AI 漫剧短集

## 口味旋钮
- **节奏**：前 3 集强钩⼦，后续每集⼀个不可逆变化
- **主角主动性**：（须点出主角名/识别点 + 每集主动 Beat 要求）
- **感情线**：（须贴合本书 CP/张力类型，可引用关键对白）
- **世界观**：少解释，多⽤事件和视觉表达（须贴合本书魔法/阶级/处境的外化方式）
- **内容边界**：不做低俗擦边，不靠恶俗标题和猎奇转化（须写清尺度与反派智商等边界）

## 禁区
- （每条必须可执行，含 不可/不得/禁止/保留 等约束，禁止空喊「要精彩」）
```

## Gate 自检（生成前请核对）

1. **模式是否说得清？** 改写档位须明确 M1/M2 或精编/深度改编/保主线/保体验。
2. **忠实对象是否明确？** 忠实对象节须写出具体保什么，不可只写「忠于原著」。
3. **口味旋钮是否冲突？** 不可同时要求「零设定倾倒」与「详细世界观解说」等矛盾取向。
4. **禁区是否可执行？** 每条禁区须能直接用于审稿，不可写空泛口号。

只输出 Markdown。
