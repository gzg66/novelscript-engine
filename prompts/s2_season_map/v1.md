# S2 季图谱 · Production Prompt v1

输出 `S2_season_map.md`，将全书章节映射为 **{season_count} 季**（以 S0 改编简报为准），章节范围必须**连续覆盖 1 至 total_chapters，无缺口、无重叠**。

## 必须遵守的 Markdown 骨架（标题字面量不可改）

```markdown
# 季图谱 · {作品名}（{season_count} 季）

## 全局主线推进（跨季骨架）
（3–5 行：每季一句阶段目标 + 哪台引擎主导）

## {season_count}季总表

| 季 | 标题 | 章节 | 季命题（主角变成谁）| 季首危机 | 季中不可回头的选择 | 季末大事件 + 下一季钩子 | 本季反派压力线 |
|---|---|---|---|---|---|---|---|
| **S1** | … | … | … | … | … | … | … |
（…共 {season_count} 行数据）

## 各季断点理由
- **S1|S2 断点（Ch…）**：须引用具体剧情节点（名场面/危机/选择），不可仅写「Ch1–30 均分」
（每处季断点：渴望被满足 + 更大威胁被打开）

## 时间线鱼骨图
```
中心骨：EP01 → EP04 → … → EP{末集}
上鱼刺：主角线 / 爱情线 / 反派线
下鱼刺：谜题线 / 名场面线 / 风险线
```
（每节点标注：原文章节来源 · 改编动作 · 情绪功能）

|---|---|---|---|

## 名场面落点
| 名场面 | 原著位置 | 落点季 | 落点集（预估）| 情绪功能 |
|---|---|---|---|---|

（覆盖故事引擎必保清单中的主要名场面）
```

## 硬性规则

- 季 ID 必须是 `**S1**`…`**S{n}**`（禁止 `Season 1`）
- 章节列用 `起始–结束` 数字格式（如 `1–30`、`112–132`）
- 季总表恰好 {season_count} 行数据 + 表头，共 8 列，每格不可空
- 按故事断点切季（非机械均分）；断点须落在「渴望被满足 + 更大威胁被打开」处
- 除末季外，「季末大事件 + 下一季钩子」列须含明确下季悬念
- 最后一季必须覆盖到第 {total_chapters} 章
- **季命题**必须写清「季末主角变成谁」，不能只是事件摘要
- 须对齐 S0 改编简报季数与 S1 主角逐季蜕变表
- 时间线鱼骨、多线推进、名场面落点三节不可省略（机考会校验标题存在）
- 名场面落点须与 S0 必保清单对应，不得凭空发明

只输出 Markdown。
